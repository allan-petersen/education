In **Log4j**, making a logger write to **multiple appenders** is straightforward. You just need to **attach both appenders to the logger**. Here’s a detailed guide for both **Log4j 1.x** and **Log4j 2.x**, since the configuration differs.

---

## **1️⃣ Log4j 2.x (Recommended)**

### **Option A: XML Configuration**

```xml
<Configuration status="WARN">
    <Appenders>
        <Console name="ConsoleAppender" target="SYSTEM_OUT">
            <PatternLayout pattern="%d [%t] %-5level %logger{36} - %msg%n"/>
        </Console>
        <File name="FileAppender" fileName="logs/app.log">
            <PatternLayout pattern="%d [%t] %-5level %logger{36} - %msg%n"/>
        </File>
    </Appenders>

    <Loggers>
        <Logger name="com.example.MyClass" level="debug" additivity="false">
            <AppenderRef ref="ConsoleAppender"/>
            <AppenderRef ref="FileAppender"/>
        </Logger>
        <Root level="info">
            <AppenderRef ref="ConsoleAppender"/>
        </Root>
    </Loggers>
</Configuration>
```

**Explanation:**

* `<AppenderRef>` lets you attach multiple appenders to a logger.
* `additivity="false"` prevents parent loggers from logging the same message again (optional).
* In this example, logs go to **console** and **file**.

---

### **Option B: Programmatically**

```java
import org.apache.logging.log4j.LogManager;
import org.apache.logging.log4j.Logger;

public class Example {
    private static final Logger logger = LogManager.getLogger(Example.class);

    public static void main(String[] args) {
        logger.info("This log will go to both appenders");
    }
}
```

> Programmatic attachment of multiple appenders is possible via `LoggerContext` and `Appender` objects, but XML/JSON/YAML configuration is simpler.

---

## **2️⃣ Log4j 1.x**

### **Option A: Properties File**

```properties
# Define appenders
log4j.appender.console=org.apache.log4j.ConsoleAppender
log4j.appender.console.layout=org.apache.log4j.PatternLayout
log4j.appender.console.layout.ConversionPattern=%d [%t] %-5p %c - %m%n

log4j.appender.file=org.apache.log4j.FileAppender
log4j.appender.file.File=logs/app.log
log4j.appender.file.layout=org.apache.log4j.PatternLayout
log4j.appender.file.layout.ConversionPattern=%d [%t] %-5p %c - %m%n

# Attach both appenders to a logger
log4j.logger.com.example.MyClass=DEBUG, console, file

# Root logger
log4j.rootLogger=INFO, console
```

**Explanation:**

* Simply list **all appenders** for the logger separated by commas.
* Log messages will be sent to **both** appenders.

---

### ✅ Key Notes

1. **Additivity**

   * In Log4j 2, `additivity="true"` (default) causes messages to also propagate to parent loggers.
   * Set `false` if you only want messages in the specified appenders.

2. **Order**

   * Logs are written to appenders in the order they are listed (usually not critical).

3. **Appender Types**

   * ConsoleAppender, FileAppender, RollingFileAppender, SocketAppender, etc., can all be combined.
